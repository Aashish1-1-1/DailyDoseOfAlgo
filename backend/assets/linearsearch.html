Understanding Linear Search in C++

What is Linear Search?

Linear search is a simple algorithm for finding a target value within an array. It works by sequentially checking each element in the array until the target is found or all elements are checked:

  1. Start from the first element of the array.
  2. Compare the target with the current element.
  3. If target matches, return its index.
  4. If target doesn't match, move to the next element.
  5. Repeat until found or all elements are checked.

Implementing Linear Search

int linearSearch(vector& arr, int target) {
    for (int i = 0; i < arr.size(); ++i) {
        if (arr[i] == target)
            return i; // Found
    }
    return -1; // Not found
}

Usage: int index = linearSearch(arr, target);

Time Complexity

  • Best Case: O(1) - target is the first element
  • Average Case: O(n) - checks each element once on average
  • Worst Case: O(n) - target at the end or not present

Space Complexity

  • Iterative: O(1) - uses only a few variables

Linear Search vs Binary Search

Aspect Linear Search Binary Search
Time Complexity O(n) O(log n)
Data Requirement Any Array Sorted Array
Best For Small or Unsorted Data Large, Sorted Data

Real-World Applications

  • Linear Data Structures: Searching in arrays, lists, etc.
  • Unsorted Databases: Linearly searching records
  • Text Processing: Searching for a substring in a text

Example Usage

int main() {
    vector arr = {12, 5, 8, 2, 16, 23, 38, 45, 56, 72, 91};
    int target = 23;

    int index = linearSearch(arr, target);
    if (index != -1)
        cout << "Found " << target << " at index " << index << endl;
    else
        cout << target << " not found" << endl;

    // Test cases
    cout << linearSearch(arr, 12) << endl;   // First element
    cout << linearSearch(arr, 91) << endl;  // Last element
    cout << linearSearch(arr, 92) << endl;  // Not in array

    return 0;
}

Conclusion

Linear search, though straightforward, can be inefficient for large datasets due to its linear time complexity. However, it remains useful in scenarios where the data is small or unsorted, or when simplicity is prioritized over efficiency.
